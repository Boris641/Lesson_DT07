# 📊 Проектирование BI-дашборда для e-commerce (Adventure Works)

🔗 **[ОТКРЫТЬ ИНТЕРАКТИВНЫЙ ДАШБОРД](https://datalens.yandex/j5hz7kc4n5n82)**

---

## 🎯 Главный фокус проекта: Понятный Data-интерфейс для бизнеса

Основная цель проекта — разработка интуитивно понятного интерфейса (Data UX) для коммерческого департамента компании Adventure Works. Дашборд спроектирован так, чтобы топ-менеджмент тратил минимум когнитивных усилий на поиск ответов и мог принимать стратегические решения на основе данных за секунды.

**Стек технологий:** Yandex DataLens (вычислительные поля, сквозная фильтрация), ETL / Очистка данных.

---

## 🎨 Принципы Data UX/UI, реализованные в проекте

При проектировании интерфейса я руководствовался принципами снижения когнитивной нагрузки:

* **Информационная иерархия (Z-паттерн):** Дашборд читается слева направо и сверху вниз. Сначала взгляд падает на три главных KPI-блока (Выручка, Прибыль, Заказы). Затем пользователь видит временную динамику и структуру продаж по макро-категориям. В самом низу находится глубокая детализация до конкретных товаров.
* **Смысловое использование цвета:** На дашборде отсутствует «цветовой шум». Для линейных и столбчатых диаграмм выбран единый нейтральный синий цвет, не отвлекающий от трендов. Цветовое кодирование применено *только* там, где это необходимо для визуального разделения категорий — на круговой диаграмме и Treemap.
* **Минимум визуального мусора:** Из чартов удалены лишние сетки, рамки и дублирующие оси. Метрики подписаны крупно и четко, а округление чисел (77.55M, 999K) делает интерфейс чистым и аккуратным.
* **Темная тема:** Контрастный темный интерфейс выбран для снижения нагрузки на глаза менеджеров при длительной ежедневной работе с мониторингом показателей.

## 🔍 Скрытая логика интерфейса (Drill-Down эффект)

Интерфейс решает классическую проблему бизнеса: *как уместить максимум информации, не перегружая экран*. Вся аналитика разделена на два связанных шага:

1. **Шаг 1. Общая картина (`Sales by categories`):** На круговой диаграмме пользователь мгновенно считывает, что 83% выручки делают велосипеды (*Bikes*). 
2. **Шаг 2. Мгновенная детализация (`Sales categories`):** Сразу под круговой диаграммой расположен Treemap (Плоская карта). Пользователю не нужно переходить на другие вкладки — он сразу видит, что внутри этих 83% локомотивами являются направления *Road Bikes* и *Mountain Bikes*, а одежда и аксессуары занимают незначительный «хвост» в углу.

## 🛠 Техническая реализация под капотом (ETL & Предобработка)

Чтобы интерфейс оставался лаконичным, отображал корректные типы данных и работал быстро, была проведена глубокая предобработка «грязных» исходных данных:

1. **Очистка финансовых метрик:** Текстовые данные о продажах и прибыли содержали знаки валют (`$`) и разделители тысяч (запятые для чисел > 1000). Написаны вложенные формулы для очистки строк и их принудительного перевода в числовой формат дробных чисел (`FLOAT`):
   ```clickhouse
   FLOAT(REPLACE(REPLACE([Cost], '\$', ''), ',', ''))
   ```
2. **Парсинг и сборка дат:** Исходные компоненты дат (день, месяц, год) хранились в виде разрозненных строк без ведущих нулей (например, `2019-3-8`), что ломало стандартное распознавание даты. Сконструировано вычисляемое поле для сборки полноценного типа данных `DATE`:
   ```clickhouse
   MAKE_DATE(INT([OrderDate_year]), INT([OrderDate_month_nbr]), INT([OrderDate_day]))
   ```
3. **Интерактивность:** Настроена сквозная интерактивная фильтрация по датам (`Order_date`) и категориям товаров (`Category`) для мгновенного пересчета всех чартов на панели.

## 💡 Ключевые бизнес-инсайты из дашборда

* **Критическая зависимость от локомотива:** Категория *Bikes* (а именно *Road* и *Mountain*) формирует более половины всей выручки компании. Маркетинговые бюджеты должны приоритетно распределяться на эти направления.
* **Оптимизация складских остатков:** Компоненты, одежда и аксессуары имеют минимальную долю в структуре продаж. Рекомендовано сократить ассортимент этих товаров до самых ходовых позиций, чтобы не замораживать оборотный капитал на складах.
* **Анализ аномалий:** Выявлен резкий изолированный пик продаж на дату `01.05.20`. Необходимо детально изучить факторы этого всплеска (маркетинговая акция, крупный оптовый заказ) для масштабирования опыта.
